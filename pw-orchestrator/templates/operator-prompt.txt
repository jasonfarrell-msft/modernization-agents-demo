Establish a modernization baseline contract for legacy-upload-demo using Playwright.

Constraints:
- Scope current user-visible behavior only.
- Do not modernize or refactor application code.
- Use the provided Azure base URL. Do not switch to localhost.
- Use GitHub Copilot custom agents to generate and refine baseline Playwright tests.
- Keep humans in approval control.
- Generate-only mode: do not execute Playwright, unit tests, npm scripts, or shell test commands.
- Use legacy-tolerant assertions. Do not assume one exact failure path.
- Do not hard-code brittle selectors tied to exact outage ids or exact form action URLs.
- Prefer resilient selectors and fallback checks for visible failure signals.

Non-negotiable generation quality rules (must be encoded in generated tests):
- Do NOT add test.skip() guards, environment variable checks, or execution blockers to generated tests.
- Generate deterministic selectors that do not trigger Playwright strict-mode ambiguity.
- For filename verification after successful upload, do not use a broad text locator that can match both flash alert and document row.
- Prefer role/semantic locators and scope file assertions to the documents table row/cell.
- Discover an upload target through user-visible navigation (Outages -> Details -> Upload action). Do not hard-code /Outages/Upload/<fixed-id>.
- Use unique file names per scenario to avoid collisions across runs.
- Build uploads as self-contained in-memory payloads (no hard dependency on external fixture files).
- For oversized uploads, accept all legacy-visible failure paths:
  - inline validation on upload page
  - redirect to details/error page with visible failure signal
  - request-level rejection (HTTP 4xx/5xx or generic large-request rejection page)
- Oversized assertions must detect request-entity-too-large style messages (including generic IIS wording) and/or request-level rejection signals.
- Assert failure signal presence before page/form-specific structure checks.

- Upload route hint: {{UPLOAD_ROUTE}}
- Allowed upload extensions: {{ALLOWED_EXTENSIONS}}
- Max upload size (MB): {{MAX_UPLOAD_MB}}

Required workflow:
1. Build a baseline scenario matrix for upload behavior.
2. Generate or update Playwright baseline tests under pw-orchestrator/playwright only.
3. For oversized upload scenarios, generate assertions that allow accepted outcomes:
   - stays on upload page with validation/error signal
   - redirects to details/error page with visible failure signal
   - request-level rejection with detectable failure signal
4. Assert failure signal presence before page/form-specific structure checks.
5. Annotate each scenario with accepted outcomes and primary/fallback assertions.
6. Include explicit anti-flake notes in generated test helpers:
   - strict-mode-safe success assertions
   - request-level oversize detection fallback
   - dynamic upload target discovery (no fixed outage id)
   - unique filename generation
7. Stop after generation. Tests will be executed post-review by running `npm test` from the pw-orchestrator/playwright folder.
8. Pause for human review.

Required output:
- Scenario matrix with priorities
- Playwright test file list
- Coverage intent by scenario (planned, not executed)
- Accepted outcomes and primary/fallback assertions by scenario
- Generated reliability safeguards (strict-mode, dynamic target discovery, and oversize-path handling)
